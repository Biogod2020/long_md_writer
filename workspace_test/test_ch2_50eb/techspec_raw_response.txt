# Technical Specification (SOTA): Chapter 2 - From Electrodes to Leads

This specification defines the architectural, editorial, and visual standards for Chapter 2. The objective is to transform abstract electrophysical concepts into a high-end, "physics-first" academic experience.

## 1. Content Guidelines (For the Writer)
- **Reasoning Style:** Employ **First-Principles Thinking**. Do not start with electrode placement; start with the **Cardiac Dipole ($\vec{P}$)**. Explain a "Lead" as a mathematical dot product: $V_{lead} = \vec{P} \cdot \vec{L}$, where $\vec{L}$ is the lead vector.
- **Technical Rigor:** 
    - **Mathematical Derivations:** Use MathJax for all equations. Explicitly derive Einthoven’s Law ($II = I + III$) and the Wilson Central Terminal (WCT) formula ($V_W = \frac{1}{3}(V_R + V_L + V_F)$).
    - **Vector Analysis:** Use "Projection" as the primary metaphor. Explain why aVR is typically negative using vector directionality relative to the depolarization wave.
- **Evidence & Mapping:** Every lead group must be mapped to its corresponding coronary artery (RCA, LAD, LCX) as per the source material. Use precise anatomical terminology (e.g., "Septal" vs. "Anterior" vs. "High Lateral").
- **Language:** Simplified Chinese (Academic tone). Use English terms in parentheses for key medical concepts (e.g., 向量投影).

## 2. Visual Design System (For the Designer)
- **Color Palette:** 
    - **Primary Base:** Slate-950 (Background), Slate-800 (Cards).
    - **Accent 1 (Vectors):** Medical Blue (`#0ea5e9`) for positive lead axes.
    - **Accent 2 (Heart):** Heart Red (`#e11d48`) for the dipole source.
    - **Accent 3 (Anatomy):** Emerald-500 for the conduction system.
- **Typography:**
    - **Body:** *Noto Serif SC* (Weight 400) for high readability.
    - **Headings:** *Inter* (Weight 700) for a modern, dashboard feel.
    - **Math/Data:** *Roboto Mono* for formulas and lead nomenclature (e.g., `aVF`, `V1`).
- **Component Styles:**
    - **Glassmorphic Callouts:** Use `backdrop-blur-md` with a `white/10` border for "Technical Deep Dives."
    - **Lead Comparison Grids:** Standardized 12-lead grid layouts using thin Slate-700 borders to mimic clinical ECG paper.

## 3. Interactive Elements (For the Transformer)
- **SVG Animations:**
    - **Vector Projection Demo:** An interactive SVG where a central heart vector (dipole) rotates, and the resulting 1D wave (sine wave amplitude) updates in real-time on a virtual Lead II axis.
    - **Hexaxial Transition:** A CSS transition that morphs the Einthoven Triangle into the Hexaxial Reference System to show their shared origin.
- **JavaScript Interactions:**
    - **The Axis Tool:** A circular slider representing the Hexaxial system. As the user slides the degree (e.g., -30° to +110°), the "Normal/LAD/RAD" status updates, and the QRS complexes for Leads I and aVF change morphology (positive/negative/isoelectric).
    - **Progressive Disclosure:** Use "Click-to-Reveal" for complex anatomical-to-lead mappings to prevent cognitive overload.

## 4. Accessibility & Performance
- **Accessibility:** Ensure all SVGs have `<title>` and `<desc>` tags. MathJax must be configured for screen reader compatibility (Speech Rule Engine). Maintain a contrast ratio of at least 7:1 for text against the dark background.
- **Performance:** 
    - **SVG Optimization:** Use inline SVGs to minimize HTTP requests.
    - **Math Rendering:** Configure MathJax for `chtml` output to ensure fast initial paint times.
    - **Lazy Loading:** Interactive JS widgets should initialize only when scrolled into view.
