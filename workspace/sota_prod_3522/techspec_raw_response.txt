# Technical Specification: Advanced Cardiovascular Electrophysiology

This specification defines the architectural and creative standards for the digital textbook "Advanced Cardiovascular Electrophysiology: From Dipoles to Clinical ECG." The goal is to create a high-fidelity, graduate-level learning environment that bridges biophysical theory and clinical practice.

### 1. Content Guidelines
The Writer must adopt a **first-principles-to-clinical-induction** reasoning style. Every clinical observation must be rooted in its underlying physics.
*   **Mathematical Rigor:** Use LaTeX for all derivations. Do not merely state the Poisson equation; demonstrate its application to the volume conductor model. Define the relationship between the current source density ($I_v$) and the potential ($\phi$).
*   **Deep-Dive Requirements:**
    *   **The T-Wave Paradox:** Explicitly explain the "double negation" logic—how the epicardium-to-endocardium repolarization sequence results in an upright vector due to the negative charge moving away from the electrode.
    *   **Vector Calculus:** Use vector notation ($\vec{p}$) for dipoles. Explain Wilson Central Terminal (WCT) as the mathematical average of the limb potentials ($V_{WCT} = \frac{1}{3}(RA + LA + LL)$).
*   **Citations:** Reference foundational electrophysiology (e.g., Einthoven, Hodgkin-Huxley) alongside SOTA clinical guidelines (ACC/AHA/ESC).

### 2. Visual Design System
The aesthetic is **"SOTA High-Tech Dark,"** mimicking high-end developer documentation (e.g., Stripe) and advanced medical imaging interfaces.
*   **Color Palette:**
    *   **Primary Background:** Deep Indigo (`#0f172a`).
    *   **Surface/Cards:** Glassmorphic Slate (`rgba(30, 41, 59, 0.7)`) with a `10px` backdrop blur.
    *   **Accents:** Depolarization Cyan (`#22d3ee`), Repolarization Amber (`#fbbf24`), and Ischemic Danger Red (`#ef4444`).
*   **Typography:**
    *   **Headings:** *Inter* (Semi-bold, tight tracking).
    *   **Body:** *Source Serif 4* for readability.
    *   **Math/Data:** *JetBrains Mono* for equations and coordinate values.
*   **Component Styles:** Callouts for "Clinical Correlation" should use a subtle left-border accent. Code blocks should be used for documenting mathematical algorithms or ion channel parameters.

### 3. Interactive Element Specifications
The Transformer must implement high-performance, reactive components using Framer Motion or GSAP.
*   **Animated SVGs:**
    *   **The Wavefront:** A synchronized animation showing the depolarization wavefront (Cyan) moving through a 2D heart silhouette while simultaneously drawing the QRS complex on a adjacent grid.
    *   **Dipole Vector:** A dynamic arrow in the center of the heart that changes magnitude and direction based on the "net" electrical activity of the ventricles.
*   **JavaScript Interactions:**
    *   **Lead Dashboard:** An interactive Einthoven’s Triangle. Clicking a lead (I, II, III, aVR, aVL, aVF) must "project" the 3D cardiac vector onto that specific axis, highlighting the resulting scalar ECG deflection.
    *   **AP Curve Slider:** A tool allowing users to manipulate Class III antiarrhythmic effects. Sliding the "K+ Blockade" variable must dynamically lengthen Phase 3 of the Action Potential and update the QT interval on a sample ECG trace.

### 4. Accessibility & Performance
*   **Accessibility:** All interactive SVGs must include `<title>` and `<desc>` tags. Color-blind friendly modes (Pattern-fill for vectors) are mandatory. Ensure contrast ratios meet WCAG AAA standards against the dark background.
*   **Performance:** All animations must be SVG-based or CSS-driven to ensure sub-100ms interaction latency. Use `IntersectionObserver` to trigger animations only when they enter the viewport to preserve CPU cycles on mobile devices.
